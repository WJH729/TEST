# 网页视频抓取工具 — 设计方案

## 1. 项目概述

一个命令行交互式工具，用于抓取网页中的视频内容（视频文件 / 流媒体）。用户通过 CLI 菜单输入目标 URL，工具自动分析页面中的视频资源，提供下载选项。

## 2. 技术选型

| 组件 | 技术 |
|------|------|
| 语言 | Python 3.10+ |
| CLI 框架 | rich + questionary（交互式菜单） |
| 页面解析 | yt-dlp（视频流解析与下载） |
| HTML 分析 | requests + beautifulsoup4 / lxml |
| 并发下载 | threading / asyncio + aiohttp |

### 核心依赖说明

- **yt-dlp**: 支持 YouTube、B站、Twitter、抖音等数百个网站的视频流提取与下载，是 youtube-dl 的活跃维护分支
- **requests + beautifulsoup4**: 对 yt-dlp 不支持的网站，手动解析页面中 `<video>` 标签、JSON 数据或 m3u8 链接
- **rich**: 带颜色/进度条的终端输出
- **questionary**: 提供下拉选择、复选框、确认等交互组件

## 3. 功能规划

### 3.1 核心功能 (MVP)

- [x] 用户输入视频页面 URL
- [x] 自动检测网站类型（yt-dlp 支持列表 / 通用网站）
- [x] 列出页面中可用的视频流（分辨率、格式、大小）
- [x] 选择特定视频流下载
- [x] 显示下载进度条
- [x] 支持自定义输出目录和文件名

### 3.2 进阶功能

- [x] 批量下载（播放列表 / 多个视频）
- [x] 仅提取音频（mp3 / m4a）
- [x] 断点续传
- [x] 代理设置
- [x] 下载历史记录
- [x] 视频格式转换（ffmpeg 集成）
- [x] 自定义 User-Agent 和 Cookie
- [x] 字幕下载

### 3.3 不实现的功能（明确不做）

- 图形界面（保持 CLI 风格）
- 自动登录/绕过付费墙
- 直播流实时录制

## 6. 模块职责

### main.py
- 初始化配置、创建输出目录
- 启动 CLI 主循环
- 全局异常捕获

### grabber/ytdlp_wrapper.py
- 封装 yt-dlp 的 `extract_info` 和 `download` 接口
- 格式化返回的视频流列表
- 提供进度回调

### grabber/page_parser.py
- 当 yt-dlp 不支持时，用 requests + BeautifulSoup 解析页面
- 查找 `<video>` 标签、JSON-LD 结构化数据、m3u8 链接
- 提取视频标题、时长、缩略图等元数据

### grabber/downloader.py
- 统一下载接口，区分 yt-dlp 下载和直接 HTTP 下载
- 支持多线程并发下载多个视频
- 断点续传逻辑

### cli/menu.py
- 使用 questionary 构建交互式菜单
- URL 输入、视频选择、目录选择
- 批量下载列表输入

### cli/progress.py
- rich.progress 自定义进度条
- 显示下载速度、剩余时间、已用时间

### utils/history.py
- JSON 文件存储下载记录
- 记录 URL、时间、文件路径、成功/失败状态

## 9. 注意事项

- yt-dlp 需要定期更新以适配网站变更，建议在工具中加入 `--update-yt-dlp` 选项
- 部分网站有反爬机制，需提供 Cookie / User-Agent 配置入口
- m3u8 格式视频需要 ffmpeg 合并分段，应在首次使用时检测 ffmpeg 是否可用
- 遵守目标网站的 robots.txt 和服务条款，仅用于个人学习用途
